EN01 🔧 Enabler — Configuração de Infraestrutura e Ambiente 3 SP
[Enabler Técnico] Configuração do repositório Git, estrutura do banco de dados PostgreSQL, projeto backend (Node.js) e frontend (React), pipeline de CI/CD básico e ambiente de homologação, para habilitar o desenvolvimento de todas as histórias funcionais da Sprint 1.
Critérios de Aceitação

  • Banco de dados PostgreSQL configurado com schema inicial (tabelas: users, projects, test_cases, modules)
  • API REST básica rodando em ambiente de homologação com health-check respondendo 200
  • Frontend React inicializado com roteamento básico e conexão com a API
  • Pipeline CI executando lint e testes unitários a cada push na branch main

US01 Cadastro de Usuário 5 SP
Como usuário novo, quero me cadastrar na plataforma informando nome, e-mail e senha, para ter acesso ao sistema e começar a gerenciar meus testes de software.
Critérios de Aceitação — Fluxo Positivo
  • Formulário exibe os campos: Nome, Sobrenome, E-mail, Senha, Nome da Empresa e Tamanho do Time
  • Ao preencher todos os campos obrigatórios e clicar em "Criar conta", o usuário é cadastrado e redirecionado para o dashboard
  • E-mail de confirmação é enviado ao usuário após o cadastro
  • Senha é armazenada com hash seguro (bcrypt) — nunca em texto puro
Critérios de Aceitação — Fluxo Negativo
  • Ao tentar cadastrar com e-mail já existente, exibe mensagem: "Este e-mail já está cadastrado."
  • Ao submeter com campos obrigatórios em branco, exibe mensagem de validação em cada campo
  • Senha com menos de 8 caracteres exibe mensagem: "A senha deve ter no mínimo 8 caracteres."

US02 Login de Usuário 3 SP
Como usuário cadastrado, quero fazer login com e-mail e senha, para acessar meus projetos e casos de teste de forma segura.
Critérios de Aceitação — Fluxo Positivo
  • Ao informar e-mail e senha corretos e clicar em "Entrar", o usuário é redirecionado para o Dashboard
  • Token JWT é gerado e armazenado para manter a sessão por 8 horas
  • Exibe saudação personalizada com o nome do usuário após o login
Critérios de Aceitação — Fluxo Negativo
  • Ao informar credenciais inválidas, exibe: "E-mail ou senha incorretos." (sem indicar qual está errado)
  • Após 5 tentativas seguidas com falha, bloqueia login por 5 minutos
  • Campos obrigatórios em branco exibem mensagem de validação antes de submeter

US04 Criar Caso de Teste 8 SP
Como QA Analyst, quero criar um caso de teste informando título, pré-condições, passos de execução e resultado esperado, para documentar formalmente o que precisa ser testado em cada funcionalidade.
Critérios de Aceitação — Fluxo Positivo
  • Formulário exibe os campos: Título (obrigatório), Pré-condições, Massa de Dados, Passos (lista ordenada), Resultado Esperado positivo e negativo, Módulo, Prioridade, Responsável e Sprint
  • Ao salvar, o CT recebe ID automático sequencial (CT-001, CT-002...)
  • CT salvo aparece imediatamente na lista de casos do módulo selecionado
  • Passos de execução podem ser adicionados dinamicamente com o botão "+ Adicionar passo"
Critérios de Aceitação — Fluxo Negativo
  • Ao tentar salvar sem título, exibe: "O título é obrigatório."
  • Ao tentar salvar sem nenhum passo de execução, exibe aviso de preenchimento

US05 Organizar Casos de Teste em Módulos 5 SP
Como QA Analyst, quero organizar casos de teste em módulos e suites dentro de um projeto, para facilitar a localização e execução por área do sistema testado.
Critérios de Aceitação — Fluxo Positivo
  • É possível criar um módulo informando apenas o nome
  • A listagem de CTs exibe uma árvore lateral com todos os módulos e a contagem de CTs em cada um
  • Ao clicar em um módulo, a lista central filtra exibindo apenas os CTs daquele módulo
  • O campo "Módulo" no formulário de criação de CT exibe os módulos existentes para seleção
Critérios de Aceitação — Fluxo Negativo
  • Não é possível criar dois módulos com o mesmo nome dentro do mesmo projeto
  • Ao tentar excluir um módulo com CTs vinculados, exibe aviso de confirmação

US12 Dashboard Básico de Qualidade 8 SP
Como QA Manager, quero visualizar um dashboard com o total de casos de teste por status (Aprovado, Reprovado, Pendente, Bloqueado), para acompanhar o progresso da sprint e a saúde do projeto em tempo real.
Critérios de Aceitação — Fluxo Positivo
  • Dashboard exibe 4 cards de métricas: Total de CTs, Aprovados, Reprovados e Pendentes
  • Cards exibem o número total e o percentual em relação ao total da sprint
  • Seção "Cobertura por Módulo" exibe barra de progresso para cada módulo com percentual de aprovação
  • Tabela "Últimas Execuções" exibe as 10 execuções mais recentes com status e responsável
  • Dashboard atualiza as métricas automaticamente quando um novo resultado é registrado
Critérios de Aceitação — Fluxo Negativo
  • Quando não há CTs cadastrados, dashboard exibe estado vazio com mensagem orientativa
